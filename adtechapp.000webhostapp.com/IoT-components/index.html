Amazon IoT

Applications

Following are the AWS IoT components, which are used to connect and transfer data to and from the devices and IoT applications.

Device gateway 

Allows secure and effectual communication through account level isolation and authorization and data transfer data to and from the devices, IoT entities such as AWS IoT Rules Engine, Device Shadows, AWS IoT applications, other AWS services and other 3rd-party services as well. It scales automatically according to usage with no operational overhead for developers and users. It supports publish and subscribe messaging pattern over secure MQTT and WebSockets and HTTPS for publishing.

 

Message broker

Offers a secure technique for devices and AWS IoT applications to publish and subscribe messages from each other. For every AWS account and region pair there is a unique topic namespace. Sending and receiving messages across AWS accounts and regions is not possible. For each session, it holds list of all client sessions and the associated subscriptions. When a message is published on a topic, it checks for sessions with subscriptions that map to the topic. Then it forwards the publish message to all sessions which are connected to client currently.

 

Rules engine

This component is responsible for message processing and integration with other AWS services. Data from message payloads is selected using an SQL-based language, then they are processed and then sent to other services, such as Amazon S3, Amazon DynamoDB, and AWS Lambda. Message Broker can be used if there is a need to republish these messages to other subscribers.

Following are the sample use cases for Rules:

• Filtering and transforming incoming messages and storing them as time series data in DynamoDB.

• Sending a push notification via SNS when the data from a sensor crosses a certain threshold.

Example for a rule:

{

"sql": "SELECT * from 'iot/tempSensors/#' WHERE temp > 50",

"description": "Rule to save sensor data when temperature is about 50",

"actions": [

{

"dynamoDB": {

"tableName": "HighTempTable",

"roleArn": "arn:aws:iam::your-aws-account-id:role/dynamoPut",

"hashKeyField": "key",

"hashKeyValue": "${topic(3)}",

"rangeKeyField": "timestamp",

"rangeKeyValue": "${timestamp()}"

}

      }

  ]

}

 

IoT Components

Security and Identity service - Provides shared responsibility for security in the AWS cloud. 

Device/Thing registry – Helps to organize and track the resources associated with each thing/device. Enables us to register the things/devices and associate its attributes. Thing type allows to fine common features for devices that are of similar type.

Device/Thing shadow – Contains properties of a connected device and responsible for cloud and mobile applications to communicate with the registered devices. Stores and retrieve current state and information for a thing in JSON format.

Device/Thing Shadows service - Allows to specify persistent representations devices in AWS cloud. When updated state information is published to thing shadow, once connected thing can synchronize its state.
