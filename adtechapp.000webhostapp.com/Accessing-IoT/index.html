Accessing AWS IoT

AWS IoT has default access to below Amazon services to enable developers to use their features:

  • Amazon Simple Storage Service - Provides scalable storage in the AWS cloud.
  • Amazon DynamoDB - Provides managed NoSQL databases.
  • Amazon Kinesis - Enables real-time processing of streaming data at a massive scale.
  • AWS Lambda - Runs your code on virtual servers from Amazon EC2 in response to events.
  • Amazon Simple Notification Service - Sends or receives notifications. For more information, see Amazon SNS.
  • Amazon Simple Queue Service - Stores data in a queue to be retrieved by applications. For more information, see Amazon SQS.

AWS IoT Device is used to optimize the memory, power, and network bandwidth consumption and also offers high-level abstractions like device synchronization.

Applications access AWS IoT using AWS SDKs available for various programming languages. It also simplifies development and operations of IoT applications by integrating with Amazon CloudWatch.

AWS IoT provides open-source device SDKs for C, Node.js, and the Arduino Yún platform to simplify the developers running the code. It also provides features to access hardware profiles connections for IoT applications.

Below options are available to connect and develop IoT applications:

1.AWS Command Line Interface (AWS CLI) - For Windows, OS X, and Linux to create and manage things, certificates, rules, and policies

2.AWS IoT API – Language specific API's

3.AWS SDKs - SDKs wraps the HTTP/HTTPS API and allow to develop in any of the supported languages

4.AWS IoT Device SDKs - For devices to send messages and receive messages from AWS IoT
